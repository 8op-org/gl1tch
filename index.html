╔══[ ABOUT ]════════════════════════════════════════╗

// orcai — abbs //

Most AI tools want you in a browser. orcai runs in your terminal, inside tmux, next to your editor and shell. Launch it, get a session. Point any CLI tool at it with a 10-line YAML file. Chain Claude, Ollama, or local models into YAML pipelines where each step knows its input and output types. In the ABBS model, widgets scroll — not pages. Your workspace is the terminal; orcai is the bulletin board.

// plugin architecture //

Any CLI tool is a plugin. Write a YAML adapter and orcai discovers it automatically. For complex orchestration, implement the gRPC sidecar interface in any language. Plugins live on your PATH — no registry, no daemon, no installation ceremony. 

╔══[ YAML ADAPTER ]══════════════════════════════════════════════════╗
name: search
binary: rg
description: ripgrep file search
args:
  - "{{query}}"
  - "{{path}}"
input:
  type: object
  properties:
    query: { type: string }
    path:  { type: string }
output:
  type: string
╚═════════════════════════════════════════════════════════════════╝
╔══[ GRPC SIDECAR ]════════════════════════════════════════════════╗
║  Implement the Plugin interface for full control:               ║
║                                                                  ║
type Plugin interface {
    Describe() PluginInfo
    Execute(ctx context.Context, input Input) (Output, error)
}
