// orcai //

Most AI tools want you in a browser. orcai runs in your terminal, inside tmux, next to your editor and shell. Launch it, get a session. Point any CLI tool at it with a 10-line YAML file. Chain Claude, Ollama, or local models into YAML pipelines where each step knows its input and output types. When you close the terminal, it's gone ‚ÄĒ no Electron process lingering, no tab to find.

‚ĒĆ‚ĒÄ TMUX SESSIONS ‚ĒÄ‚Ēź

orcai tracks your AI workspaces as tmux sessions in a sidebar. Spawn one per task, switch mid-pipeline, kill what you're done with. Everything lives in the terminal you already have open.

‚ĒĆ‚ĒÄ PLUGIN SYSTEM ‚ĒÄ‚Ēź

Write a YAML file that points at any CLI tool ‚ÄĒ rg, jq, ffmpeg, your own scripts ‚ÄĒ and it's a pipeline node. Need something more complex? Implement the gRPC interface in Go.

‚ĒĆ‚ĒÄ AGENT PIPELINES ‚ĒÄ‚Ēź

YAML pipelines connect AI providers to plugins in sequence. Each step has typed input and output schemas, so you can chain a Claude prompt into a jq filter into a file write without any glue code.
