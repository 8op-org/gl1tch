The issue describes a need for improving error reporting and resolution in an application. Specifically, the user requests that errors should include:

1. A short explanation of what happened.
2. Guidance on how to resolve the issue or report it.
3. A way to link the error to a system (such as logging).

Based on the evidence provided, these are the key points discussed:

- The current state lacks clear explanations and guidance for users facing errors.
- There is no mechanism to track user-reported issues internally.

The proposed solution involves enhancing the error messages to include more detail and pointers for further action.
