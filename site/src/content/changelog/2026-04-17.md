### Workflow Engine

- **S-expression DSL overhaul**: All `.glitch` workflows now use s-expr syntax with `~ref` and `~(form)` interpolation, `def` bindings, and a full expression evaluator (`evalForm`, `renderQuasi`)
- **New control-flow forms**: `(par ...)` for concurrent step execution, `(when)`/`(when-not)` conditionals, `(filter)`, `(reduce)`, `(->)` threading macro, `(retry)`, `(timeout)`, `(let)`, `(catch)`, `(cond)`, `(map)`
- **Phases and gates**: Workflows can declare `(phase ...)` blocks with `(gate ...)` pass/fail evaluation to guard execution
- **Compare workflows**: `(compare ...)` / `(branch ...)` / `(review ...)` forms run parallel branches with a judge LLM that selects a winner; `--variant`, `--compare`, and `--review-criteria` CLI flags exposed
- **CLEAR framework**: Structured evaluation criteria applied to compare/batch workflows
- **Elasticsearch DSL forms**: `(search)`, `(index)`, `(delete)`, `(embed)`, `(flatten)`, `(index :upsert false)` for dedup; `(search :sort)` and `(search :ndjson)` parameters
- **`(call-workflow ...)` form**: Invoke other workflows from within a workflow with cycle detection and nested run parenting
- **`(map-resources ...)` form**: Iterate over workspace resources in a step
- **`(par ...)` parallel execution**: Steps inside a `(par)` block run concurrently; data race on steps map fixed
- **Artifacts**: Steps now record artifacts; `run_id` is available in templates; artifacts aggregated into run metadata
- **Workflow metadata**: `(tags ...)`, `(author ...)`, `(version ...)`, `(created ...)`, `(arg ...)`, `(input ...)` top-level forms now parsed and stored
- **Implicit ref extraction**: `~param.X` and `~input.X` refs are auto-extracted and merged with declared args
- **Workflow-specific help**: `glitch run <workflow> --help` renders cobra-idiomatic help from declared args
- **Better error messages**: `UndefinedRefError` includes source file/line/col and nearest-match suggestions for dotted-path misses; `env.X` and `resource.X.Y` fail loudly on missing keys
- **`(or ...)` short-circuit**: Optional refs no longer hard-fail when a key is absent
- **Namespaced plugin shorthand**: `github/prs` desugars to `(plugin "github" "prs")`
- **Smart routing (RunSmart)**: Auto-escalation loop — local → free → paid — with structural pre-validation and self-eval scoring
- **Schema drift detection**: Stale SQLite schemas (pre-1.0) are auto-detected and recreated; `steps.artifacts` column included in drift checks
- **`map` step**: Supports multiple body steps with combined output stored under body IDs; cross-step references aliased within iterations

### Providers

- **LM Studio**: Full provider with model availability check, auto-download with progress polling, load-state logging, and tiered runner integration
- **OpenAI-compatible provider**: Generic client supporting named endpoints via `provider_config` in config; resolver wired into pipeline runner and tiered runner
- **Ollama embed**: New `EmbedOllama` for `/api/embeddings` endpoint
- **Tiered escalation**: Three-tier fallback chain (local → free paid → paid) with token tracking and cost estimation
- **Self-evaluation**: Per-criterion PASS/FAIL confidence scoring parsed from review output; structural validation before full eval
- **Model updates**: Local tier now uses `lm-studio/qwen3-8b`; free tier updated to `qwen3-coder:free` (Gemma rate-limited)
- **Latency and cost tracking**: LLM telemetry includes latency, token counts, and cost per step

### CLI

- **Workspace commands**: `glitch workspace init/use/list/status/register/unregister/sync/pin/add/rm` — full workspace lifecycle management
- **`glitch run` at top level**: `glitch run <workflow>` replaces `glitch workflow run`; workflow list/new/gui moved under workspace
- **Workspace resources**: `add`/`rm` manage git, local, and tracker resources; `sync` fetches and `pin` locks with format-preserving writes
- **`--workspace` flag**: Persistent root flag scopes all commands to a workspace
- **`--set key=value`**: Pass arbitrary params to any workflow run
- **`--tag` filter**: `glitch workflow list --tag <tag>` filters by workflow tag
- **`glitch plugin`**: `list`, `help`, and dynamic subcommand dispatch for installed plugins
- **`glitch version` / `--version`**: Version flag and subcommand now work correctly
- **`glitch index`**: ES code search indexing command
- **`glitch up`**: Seeds default Kibana dashboards automatically on startup
- **Issue/PR URL routing**: Bare refs and URLs auto-routed to the appropriate workflow with parsed params
- **`--compare` batch mode**: Multi-issue, multi-variant parallel batch runs with cross-review scoring

### Plugins

- **Plugin execution engine**: Load `plugin.glitch` manifest, parse `(arg ...)` forms with defaults/validation, run sub-workflow
- **Plugin discovery**: Searches local and global plugin directories
- **SDK forms**: `json-pick`, `lines`, `merge`, `http`, `read-file`, `write-file`, `glob` available inside plugin workflows
- **Workspace-aware plugins**: Plugin resolution respects active workspace
- **Plugin cache and shared step seeding**: Plugins share a step cache across calls within a run
- **`ArgDef` improvements**: `Example` and `Implicit` fields added for richer help output
