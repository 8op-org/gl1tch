### Workflow Engine

- **S-expression syntax** is now the primary workflow format (`.glitch` files). All built-in workflows have been migrated.
- **`(par ...)`** runs steps concurrently inside a phase, with a mutex-safe execution context.
- **`(phase ...)` and `(gate ...)`** forms partition workflows into named phases with pass/fail gates.
- **`(compare ...)`, `(branch ...)`, `(review ...)`** run parallel model variants and pick a winner via a judge prompt. Supports `--variant`, `--compare`, and `--review-criteria` CLI flags.
- **`(call-workflow ...)`** executes another workflow inline with cycle detection and nested run tracking.
- **`(->) ` threading macro** pipes output between forms without intermediate bindings.
- **`(reduce)`, `(filter)`, `(when)`, `(when-not)`** collection and conditional control-flow forms added.
- **`(flatten)`** converts a JSON array step to NDJSON for downstream processing.
- **`(input ...)` and `(arg ...)`** declare workflow inputs; implicit `~param.X` refs are auto-extracted. Source locations are threaded into error messages.
- **`glitch run <workflow> --help`** prints cobra-idiomatic, workflow-specific help derived from `(arg ...)` declarations.
- **`(map-resources ...)`** iterates over workspace resources inside a workflow.
- **`(index :upsert false)`**, **`(search :sort)`**, **`(search :ndjson)`**, **`(llm :format "json")`** new step parameters for ES and LLM forms.
- **String and JSON builtins** — `upper`, `lower`, `trim`, `split`, `join`, `pick`, `assoc` available as template functions inside workflows.
- **`(def ...)`** bindings and symbol resolution in workflow scope.
- **CLEAR framework** for structured compare/batch workflows with review criteria.
- **Smart routing** (`RunSmart`) — tiered LLM escalation (local → free-paid → paid) with structural validation and self-eval scoring.
- **Parallel batch runs** execute workflow variants concurrently per issue via goroutines.

### Providers

- **LM Studio** provider added with model auto-download, progress polling, and load-state logging.
- **OpenAI-compatible** provider supports named endpoints via `[providers]` config entries.
- **Ollama embed** provider added for `/api/embeddings`.
- **Self-evaluation** and **structural validation** hooks added to provider interface for smart routing.
- **Free model updated** to `qwen3-coder:free` (gemma rate-limited); local tier now uses `lm-studio/qwen3-8b`.

### CLI

- **`glitch run <workflow>`** promoted to top-level command; `workflow run` removed.
- **`glitch workspace`** — full workspace management: `init`, `use`, `list`, `status`, `register`, `unregister`, `sync`, `pin`, `add`, `rm`.
- **`glitch plugin`** — `list`, `help`, and dynamic subcommand dispatch for installed plugins.
- **`glitch observe`** — natural-language query over indexed knowledge.
- **`glitch index`** — index a repo into Elasticsearch for code search.
- **`glitch up`** auto-seeds default Kibana dashboards on first run.
- **`glitch --version`** / **`glitch version`** now work correctly.
- **`--set key=value`** passes arbitrary params to workflow run.
- **`--tag`** filters `workflow list` results.
- **`--workspace`** persistent root flag scopes all commands to a named workspace.
- **Issue/PR URL routing** — bare refs and GitHub URLs are automatically dispatched to the appropriate workflow.

### Plugins

- **Plugin discovery** from local (`.glitch/plugins/`) and global directories.
- **Plugin manifests** (`.glitch` files) declare metadata, shared defs, and typed `(arg ...)` specs with defaults and validation.
- **SDK forms** available in plugins: `json-pick`, `lines`, `merge`, `http`, `read-file`, `write-file`, `glob`.
- **Namespaced shorthand** — `github/prs` desugars to `(plugin "github" "prs")`.
- **`ArgDef`** gains `Example` and `Implicit` fields surfaced in plugin help output.
- Plugins are now **workspace-aware** — they resolve resources from the active workspace.

### GUI

- **Full redesign** — Monaco editor replaces CodeMirror, ELK-based graph layout, breadcrumb navigation, token/cost display.
- **Workflow runs** — list, detail, run-tree view (`/api/runs/:id/tree`), children expansion, parent/child run linking.
- **Workspace resources** — CRUD, sync, and pin endpoints; workspace switcher in settings.
- **Workflow list** enriched with tags, author, version metadata; run list enriched with model, tokens, and cost.
- **Actions API** — `(action ...)` keyword exposes workflow actions as API endpoints.
- **Kibana iframe** endpoints for per-workflow and per-run telemetry panels.
- **Single binary** — frontend is bundled and embedded via `go:embed`; no separate server needed.
- **`glitch syntax`** highlighting in the editor.

### Code Intelligence

- **Tree-sitter indexer** extracts symbols, edges, and cross-file import/call/containment relationships.
- **`IndexRepoGraph`** pipeline indexes a full repo into Elasticsearch (Symbols + Edges indices).
- **Graph BFS traversal** in the observer with a `--depth` flag to control traversal depth.
- **`LearningDoc`** type tracks phase and objective fields for learning-over-time charts.
- **`TermsAgg`** method enables incremental indexing lookups without full re-index.

- **New doc pages**: Providers, Batch Runs, Code Intelligence, Workspaces.
- **Homepage redesign** with updated hero, how-it-works section, and feature highlights.
- **`site-sync` workflow** — single command to sync changelog, labs, and rebuilt assets.
- **Playwright tests** gate every site build; labs pages have dedicated coverage.
- All docs and examples updated to **s-expr unquote (`~`) interpolation syntax**.
